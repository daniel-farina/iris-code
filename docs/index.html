★What's new

v0.3.8 brings a search-first agent prompt, a 4000-line read default, 128K-token context, persisted resume, and a refreshed mood-pigmy launch banner. The setup wizard now clones MTPLX upstream main directly — all eight perf PRs landed.

  • hip --resume picker. Arrow-key list of recent sessions (cap 100, 10-row scroll window, two-line entries with full cwd path). Selecting a row reloads its full conversation history — not just the session id, so the model picks up where it left off.
  • Queued follow-ups. Press Enter while a turn is streaming to stage a follow-up message; the agent auto-chains through the queue between turns instead of dropping it.
  • Effectively uncapped --max-tokens. Default raised to 128K to match the new 128K context window; the agent auto-continues on finish_reason=length truncation so long generations resume seamlessly.
  • Search-first system prompt. Rewritten DEFAULT_SYSTEM_PROMPT around an explicit search-first playbook. A/B against an open-ended task: 2× faster wall time, 2.7× fewer output tokens, 4× more surgical edits — with no regression on targeted edits.
  • 4000-line read default. Was 200; bumped so the model gets the whole file on a typical 3500-line source instead of a truncated head + an early give-up. MAX_BYTES=1MiB still bounds the absolute worst case.
  • Mood-pigmy launch banner. Refined truecolor mascot art (mascot name: mood) replaces the prior pigmy iteration. Same 14-line ANSI-truecolor format.
  • MTPLX upstream main. The first-run wizard now clones youssofal/MTPLX#main directly — all eight perf PRs we were carrying landed. MTPLX_SESSION_BANK_* env vars (per_session 16G, max 32G, max_entries 24) are still set before spawn as per-instance tuning.
  • Slash & colon commands. Both /help and :help work; tab-completes after typing / or :; the empty buffer shows an inline hint of the most useful next move.
  • /new vs /reset. /new starts a fresh conversation - clears history and rotates the MTPLX session id so the prefix cache resets. /reset keeps the session id so you stay on a warm cache.
  • /context usage tracker. Shows current tokens vs the 128K window with input/completion split and turn count. After every response a one-line bar prints inline: ─ ctx 12.4K/128K (10%) · 3 turns ─, colored green → warn → bad past 75% / 90%.
  • Sticky bottom bar, resize-aware. The live tok / lines / tok-s / ttft bar survives terminal resizes (SIGWINCH-aware) and falls back gracefully on dumb terminals.
  • Hippo-purple truecolor theme. Default theme + bundled hippo logo. Double-Ctrl-C exits; bare exit, quit, bye exit with confirmation.
  • First-run setup wizard. Probes for MTPLX, offers to clone & install, downloads the model, starts the server in the background, then drops you straight into chat. Auto-detects Python < 3.11 and offers brew install python@3.12; recreates stale .venv built with old Python; surfaces real pip errors inline.
  • In-place self-update. hip --update downloads and replaces the binary from the latest GitHub release - no curl pipe, no Cargo rebuild.

2What it does

A tight subset of opencode's agent loop, without the parent-agent prompt overhead.

  • 10 built-in tools. read, grep, edit, bash, list, glob, search, diff, tree, peek_log. Each is a JSON-schema spec sent to the model plus a small local executor. Gitignore-aware where it matters.
  • Thinking mode shown by default. <think> blocks render in a dim panel above the response. Hide with --hide-thinking or MLX_CODE_SHOW_THINKING=0; verbose with -v to also get full tool output and stats.
  • Dry run. --dry-run makes edit and bash return (dry_run) would… previews instead of writing or executing - the agent loop still runs end-to-end so you can validate plans safely.
  • Sticky bottom status bar. ANSI scroll regions (DECSTBM) pin a live tok / lines / tok-s / ttft / total bar to the bottom row while output streams above. Falls back to inline \r if the terminal can't, or with MLX_CODE_NO_STICKY=1.
  • ~1.35K token prompt overhead vs opencode's ~13K - roughly 9.6× lighter cold prefills. Run hip --inspect-prompt for a live per-tool breakdown.

4Architecture

No daemon, no TUI framework, no MCP. Just modules.

Core (src/*.rs)

ModuleResponsibility
main.rsCLI flag parsing, mode dispatch (one-shot, agent, chat, smoke, summary).
agent.rsTool-using agent loop — up to 30 rounds of stream → tool_calls → tool results.
client.rsThin HTTP/SSE client for the OpenAI-compatible MTPLX endpoint.
schema.rsSerde types for the chat-completions wire shape (only fields we use).
repl.rsrustyline Helper: :command completion, path completion, history.
pretty.rsPretty-print streamed think blocks, content, tool calls, tool results.
theme.rsriver / light / mono color themes — named accessors over 256-color ANSI.
logo.rsStartup ASCII logo: 6-row "HIPPO" with hippo glyph + river-blue gradient.
sticky_bar.rsSticky bottom status bar via DECSTBM scroll regions, with safe fallback.
sparkline.rsTiny Unicode-block sparkline + small formatting helpers.
runlog.rsAppend-only JSONL run log at ~/.mlx-code/logs/runs.jsonl.
read_cache.rsProcess-global content cache for the read tool within one run.
dry_run_log.rsAccumulator of paths a dry run would have mutated; printed at end-of-run.
setup.rsFirst-run wizard: probes for MTPLX, clones youssofal/MTPLX#main, installs deps, downloads the model, spawns the server, then drops into chat.
session_store.rsPersists conversation history per session id and reloads it on --resume; not just rotating the session id.
typeahead.rsPending-message queue: /queue add <msg> stages follow-ups; the chat loop auto-dequeues each as the next prompt.
updater.rship --update: fetches the latest GitHub release binary and replaces the running executable in place.

Tools (src/tools/*.rs)

ModuleTool
tools/mod.rsTool registry — JSON-schema specs + dispatch to local executors.
tools/read.rsread(path, offset?, limit?, around?, context=20); refuses >1MB.
tools/grep.rsgrep(pattern, path?, glob?) via the ignore crate, 50-hit cap.
tools/edit.rsedit(path, old, new, replace_all?, dry_run?); empty old creates.
tools/bash.rsbash(command, timeout_s?) in zsh -lc, 16 KB output cap.
tools/list.rslist(path) directory listing.
tools/glob.rsglob(pattern, path?), gitignore-aware, 200-path cap.
tools/search.rssearch(query, …) with query expansion + definitions/files-only modes.
tools/diff.rsdiff(a, b, context=3) LCS unified diff with @@ hunks.
tools/tree.rstree(path?, depth=2, show_hidden?), gitignore-aware, 500 entries.
tools/peek_log.rspeek_log(n=10, failed_only?) — last N rows from runs.jsonl.
